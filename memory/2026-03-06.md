## Root Cause Analysis: 504 Gateway Timeout Errors — March 6, 2026

### Problem
Dashboard consistently returns 504 Gateway Timeout errors, even after backend restarts.

### Root Cause Identified
**The Node.js proxy server (port 4173) keeps dying while the backend (port 8000) stays running.**

### Infrastructure Chain
```
Browser → ngrok (HTTPS) → Proxy (port 4173) → Backend (port 8000)
              ↑______________↑
                     BROKEN LINK
```

### Why It Happens
1. Backend API (Python/uvicorn on port 8000) starts successfully
2. Proxy server (Node.js on port 4173) crashes silently after some time
3. ngrok tunnel can't reach backend through dead proxy
4. All API calls return 504 Gateway Timeout

### Symptoms
- Backend health check works: `curl http://localhost:8000/health`
- Proxy health check fails: `curl http://localhost:4173/api/email-templates/`
- ngrok URL returns 504 for all `/api/*` routes

### Immediate Fix
```bash
cd /data/workspace/frontend && node proxy-server.cjs &
```

### Permanent Solutions (To Implement)

**Option 1: PM2 Auto-Restart (Recommended)**
```bash
npm install -g pm2
pm2 start proxy-server.cjs --name "dashboard-proxy"
pm2 startup
pm2 save
```

**Option 2: Systemd Service**
Create `/etc/systemd/system/dashboard-proxy.service`:
```ini
[Unit]
Description=Dashboard Proxy Server
After=network.target

[Service]
Type=simple
User=openclaw
WorkingDirectory=/data/workspace/frontend
ExecStart=/usr/bin/node proxy-server.cjs
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

Then:
```bash
sudo systemctl enable dashboard-proxy
sudo systemctl start dashboard-proxy
```

### Key Learning
When debugging 504 errors, always verify the entire chain:
1. Is backend running? (port 8000)
2. Is proxy running? (port 4173)
3. Is ngrok tunnel active? (check localhost:4040)

Don't assume backend = working system.
