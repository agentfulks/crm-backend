# LUCAS DAILY CHECKLIST
## VC Outreach Engine — 15-Minute Daily Routine
**Purpose:** Maintain approval velocity and pipeline flow  
**Time Required:** 15 minutes maximum  
**Frequency:** Every morning, non-negotiable

---

## THE CHECKLIST

### Part 1: Board Scan (2 minutes)

```
□ Open Trello board
□ Check "Awaiting Approval" count
□ Check "Approved/Send" count  
□ Check "Follow-up" for replies
```

**Decision Gate:**
- If Awaiting Approval > 10 → Jump to Part 3 (Batch Mode)
- If Awaiting Approval ≤ 10 → Continue to Part 2

---

### Part 2: Daily Approval (10 minutes)

**Goal:** Process ≤ 5 cards from Awaiting Approval → Approved/Send

```
□ Open first card in Awaiting Approval
□ Scan research summary (30 sec)
□ Make decision: APPROVE or REJECT (15 sec)
□ If APPROVE: Move to Approved/Send
□ If REJECT: Move to Archive with reason
□ Repeat for up to 5 cards
```

**Decision Rules:**
- **APPROVE if:** Tier 1-2 fit, research complete, messaging ready
- **REJECT if:** No thesis fit, incomplete research, wrong stage
- **No MAYBE:** Every card moves forward or gets archived

**Time Allocation:**
- 2 minutes per card maximum
- 5 cards × 2 min = 10 minutes
- If decision takes longer than 2 min, approve and iterate later

---

### Part 3: Queue Maintenance (3 minutes)

```
□ Check "Approved/Send" count
□ If > 20 cards → Execute sends today (separate block)
□ If 10-20 cards → Schedule send block tomorrow
□ If < 10 cards → Research team can add more
```

**Pipeline Health Indicators:**

| Metric | Healthy | Warning | Critical |
|--------|---------|---------|----------|
| Awaiting Approval | 0-5 | 6-10 | >10 |
| Approved/Send | 10-20 | 21-30 | >40 |
| Follow-up | 5-15 | 16-25 | >30 |

**Actions by Status:**
- **Healthy:** Continue normal cadence
- **Warning:** Increase approval time to 20 min
- **Critical:** Emergency batch session (45 min)

## WEEKLY RHYTHM

### Monday (Review Day)

```
□ 5 min: Review weekend replies
□ 15 min: Approval block (per checklist)
□ 5 min: Plan week's send volume
```

### Tuesday-Thursday (Standard Days)

```
□ 15 min: Approval block (per checklist)
□ 30-45 min: Send execution (separate block)
```

### Friday (Follow-up Day)

```
□ 10 min: Check all replies
□ 15 min: Send follow-ups
□ 5 min: Weekly metrics review
```

---

## EMERGENCY PROTOCOLS

### Trigger: Awaiting Approval > 15 Cards

```
1. Pause all new research
2. Block 45 minutes same day
3. Batch-approve entire queue
4. Resume normal cadence
```

### Trigger: Approved/Send > 50 Cards

```
1. Priority shift to send execution
2. Reduce approval time to 5 min
3. Execute 10+ sends/day until < 30
```

### Trigger: Follow-up > 30 Cards

```
1. Dedicate Friday to follow-ups only
2. Archive non-responders after 3 attempts
3. Re-prioritize warm leads
```

## REMINDERS

1. **15 minutes is the maximum, not the goal.** If queue is 0, you're done in 2 minutes.
2. **Batch decisions are faster than individual deliberation.** Trust the research quality.
3. **Approval velocity > research velocity.** Never let the queue accumulate.
4. **This is a constraint management task, not a deep work task.** Move fast, decide faster.
