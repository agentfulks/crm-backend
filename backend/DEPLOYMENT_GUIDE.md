## Quick Start Options

### Option 1: Render (Recommended - Free Tier)

1. **Create Postgres Database:**
   - Go to https://dashboard.render.com/new/database
   - Name: `vc-outreach-db`
   - Region: US East (or closest to you)
   - Plan: Free
   - Click "Create Database"
   - Copy the "Internal Database URL" (starts with `postgresql://...`)

2. **Create Web Service:**
   - Go to https://dashboard.render.com/new/web-service
   - Connect your GitHub repo or use "Deploy from Docker"
   - Name: `vc-outreach-api`
   - Region: Same as database
   - Plan: Free
   - Environment Variables:
     ```
     DATABASE_URL=postgresql://... (from step 1)
     SECRET_KEY=your-secret-key-here (generate: openssl rand -hex 32)
     ENVIRONMENT=production
     ```
   - Build Command: `pip install -r requirements.txt && alembic upgrade head`
   - Start Command: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`

3. **Deploy:**
   - Click "Create Web Service"
   - Render will build and deploy automatically

### Option 2: Railway (Alternative)

1. **Deploy via Template:**
   - Go to https://railway.app
   - Click "New Project" → "Deploy from GitHub"
   - Select your repo
   - Railway auto-detects Python and creates Postgres

2. **Environment Variables:**
   - `DATABASE_URL` (auto-generated by Railway)
   - `SECRET_KEY` (generate and add manually)
   - `ENVIRONMENT=production`

### Option 3: Supabase (Postgres Only)

1. **Create Project:**
   - Go to https://supabase.com
   - Create new project
   - Go to Settings → Database
   - Copy connection string

2. **Run Migrations Locally:**
   ```bash
   export DATABASE_URL="postgresql://..."
   cd backend
   alembic upgrade head
   ```

## Local Development (Docker)

If you have Docker access:

```bash
cd backend
docker-compose up -d
```

This starts:
- Postgres on port 5432
- Adminer (DB UI) on port 8080
- Backend API on port 8000

**Note:** Current sandbox lacks Docker. Use cloud options above.

## Environment Variables

Create `.env` file:

```bash
# Database
DATABASE_URL=postgresql://user:pass@host:5432/dbname

# Security
SECRET_KEY=your-secret-key-here

# Environment
ENVIRONMENT=development  # or production

# Trello (for automation)
TRELLO_API_KEY=your-trello-api-key
```

## API Endpoints

Once deployed, the API provides:

| Endpoint | Description |
|----------|-------------|
| `GET /health` | Health check |
| `GET /api/funds` | List all VC funds |
| `GET /api/packets` | List outreach packets |
| `POST /api/packets` | Create new packet |
| `GET /api/contacts` | List contacts |
| `GET /api/interactions` | List interactions |
| `GET /api/outreach` | List outreach attempts |
| `GET /api/meetings` | List meetings |
| `GET /api/notes` | List notes |

Full API docs available at `/docs` (Swagger UI) when deployed.
