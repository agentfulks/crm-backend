## Backend API Stability Issues — March 5, 2026 (Evening)

### Problem
Backend API repeatedly crashing or becoming unresponsive, causing 504 Gateway Timeout errors on frontend.

### Root Causes Identified
1. **Pydantic UUID serialization errors** in email_templates endpoints
2. **Backend process dying** silently without restart
3. **Proxy server** losing connection to backend

### Solutions Applied

**1. Fixed Email Template Creation (Pydantic UUID Issue)**
Changed from Pydantic model validation to plain dict returns in `/data/workspace/backend/app/api/routes/email_templates.py`:
```python
# Before: Returned Pydantic model (failed on UUID→string conversion)
return EmailTemplateRead.model_validate(template)

# After: Return plain dict
return {
    'id': str(template.id),
    'name': template.name,
    ...
}
```

**2. Fixed POST Endpoint Data Format**
Changed from JSON body to query parameters to avoid Pydantic validation issues:
```python
# Backend accepts query params
POST /api/email-templates/?name=X&subject=Y&body=Z
```

**3. Monitoring Processes**
Backend requires active monitoring - when it dies:
```bash
pkill -f uvicorn
nohup .venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000 > /tmp/backend.log 2>&1 &
```

### Dashboard Infrastructure
Current setup requires 3 running processes:
1. **Backend API** (port 8000) - Python/FastAPI
2. **Proxy Server** (port 4173) - Node.js/Express
3. **ngrok Tunnel** - HTTPS endpoint

### Learning
- Pydantic v2 with SQLAlchemy UUID columns requires careful serialization
- Query params more reliable than JSON body for simple CRUD
- Always verify backend health before debugging frontend issues
